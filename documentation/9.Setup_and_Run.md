# 🛠️ Setup and Run Guide

This interactive guide walks you through setting up and launching the **Keya** system on your local workstation.

---

## 📋 System Prerequisites

Ensure your system meets the following standard software requirements:

*   **Node.js**: `v18.0.0` or higher (we highly recommend `v20.x` or higher LTS releases)
*   **npm**: `v9.0.0` or higher (automatically installed with Node.js)
*   **Web Browser**: A modern web browser with **Web Speech API** support (Google Chrome, Microsoft Edge, Opera, or Safari) for voice searches and speech synthesis.

---

## ⚡ Step-by-Step Installation & Run

Follow these terminal commands sequentially to bootstrap and launch the web interface:

### 1️⃣ Navigate to Project Workspace
Open your desktop terminal and enter the project folder:
```bash
cd project
```

### 2️⃣ Install Core Dependencies
Install Vite, Tailwind CSS, React, and Lucide React packages:
```bash
npm install
```

### 3️⃣ Start Development Environment
Launch Vite's high-speed local dev server:
```bash
npm run dev
```
🎉 **Success!** Your terminal will output the local network address:
```text
  VITE v7.2.4  ready in 250 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
  ➜  press h + enter to show help
```
Open **`http://localhost:5173/`** in your browser to experience the dictionary!

---

## 🚀 Build and Production Deployment

Keya uses a single-file compiler plug-in to build highly portable static output files.

### 📦 Compile a Production Bundle
To build an optimized production build:
```bash
npm run build
```
This bundles and inlines all Javascript modules, styles, and assets into a **single, portable `index.html` file** inside the `dist/` directory:
```text
dist/index.html   ~333 kB
```

### 🔍 Preview the Built Application
To preview your compiled build locally:
```bash
npm run preview
```

### 🌐 Direct Deployment
*   **Zero-Config Deployments**: The single file inside `dist/index.html` is fully complete and self-contained. You can drop this single file into **Netlify**, **Vercel**, **GitHub Pages**, or any static web host, and it will work instantly with no server configuration needed!
*   **Netlify Ready**: A pre-configured `project/netlify.toml` file is already in the project directory for seamless Netlify git-push deploys.

---

## 🔌 Online Fallback Integration Note
When translating text offline, Keya requires zero network traffic. If a searched word or sentence does not match any offline keys, Keya instantly fallback-queries the **MyMemory free Translation API** over HTTP. To ensure these fallback lookups succeed, your machine should have an active internet connection. If offline, Keya gracefully returns the word-by-word structural layout.
