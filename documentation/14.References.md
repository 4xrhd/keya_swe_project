## 1. Core Web Technologies & Frameworks

This section references the libraries, languages, and compile tools selected to build Keya's lightweight, high-performance, single-bundle architecture.

*   **[R1] React 19 (Component UI Framework)**
    *   *Reference:* Meta Open Source, "React – A JavaScript library for building user interfaces," 2026.
    *   *Application:* Used for structured, reactive user interface components, robust state hooks (`useState`, `useEffect`, `useRef`), and high-frequency concurrent typing responsiveness.
    *   *URL:* [react.dev](https://react.dev)

*   **[R2] TypeScript 5 (Static Type Safety)**
    *   *Reference:* Microsoft Corporation, "TypeScript: JavaScript with syntax for types," 2026.
    *   *Application:* Ensures total frontend type-safety, strong schemas for dictionary nodes, explicit function signatures for translation fallbacks, and compile-time prevention of null pointer exceptions.
    *   *URL:* [typescriptlang.org](https://www.typescriptlang.org)

*   **[R3] Vite 7 & Bundler Optimization**
    *   *Reference:* E. You et al., "Vite: Next Generation Frontend Tooling," 2026.
    *   *Application:* Provides rapid development compilation using ES Modules and compiles the production assets into a single optimized distribution bundle via specialized plugin integrations.
    *   *URL:* [vite.dev](https://vite.dev)

*   **[R4] Single-File Asset Bundle Compilation**
    *   *Reference:* R. Karr, "vite-plugin-singlefile: Bundle all build assets into a single HTML file," npm Registry, 2025.
    *   *Application:* Crucial compile-time optimization that inlines all JavaScript modules, base64 assets, and Tailwind styles into `dist/index.html` to guarantee instant, offline portability.
    *   *URL:* [npmjs.com/package/vite-plugin-singlefile](https://www.npmjs.com/package/vite-plugin-singlefile)

*   **[R5] Tailwind CSS v4 (Utility-First Design System)**
    *   *Reference:* Tailwind Labs Inc., "Tailwind CSS: A utility-first CSS framework for rapid UI development," 2026.
    *   *Application:* Powers the application's clean styling, consistent spacing scale, dark-mode styling matrix, responsive layout, and transitions.
    *   *URL:* [tailwindcss.com](https://tailwindcss.com)

## 2. Natural Language Processing, Lemmatization & Stemming

This section documents the linguistic and algorithmic research behind the custom lemmatizer engine that successfully maps irregular tenses and plurals back to their dictionary base lemma.

*   **[R6] Algorithmic Suffix Stripping & Stemming**
    *   *Reference:* M. F. Porter, "An algorithm for suffix stripping," *Program: electronic library and information systems*, vol. 14, no. 3, pp. 130-137, 1980.
    *   *Application:* Guided the algorithmic rules for stripping common English plurals (e.g., `-es`, `-ies`, `-s`) and verb inflections (`-ing`, `-ed`) to optimize base-word extraction in under 5ms.
    *   *DOI:* [10.1108/eb011812](https://doi.org/10.1108/eb011812)

*   **[R7] Computational Lemmatization & Lexicons**
    *   *Reference:* D. Jurafsky and J. H. Martin, *Speech and Language Processing: An Introduction to Natural Language Processing, Computational Linguistics, and Speech Recognition*, 3rd ed. draft, Chapter 2: "Regular Expressions, Text Normalization, and Edit Distance", 2024.
    *   *Application:* Formed the theoretical basis for mapping irregular lexical mutations (e.g., past participles like `"went"` $\rightarrow$ `"go"`, or irregular plurals like `"women"` $\rightarrow$ `"woman"`) using structured offline lookup dictionaries.
    *   *URL:* [web.stanford.edu/~jurafsky/slp3/](https://web.stanford.edu/~jurafsky/slp3/)

## 3. Web Standards & Native Browser APIs

This section references the open standards and browser APIs leveraged to provide voice input and pronunciation playback fully client-side.

*   **[R8] W3C Web Speech API (Voice Synthesis & Recognition)**
    *   *Reference:* W3C Web Speech API Draft Specification, "Web Speech API," Web Incubator Community Group (WICG), Jan. 2023.
    *   *Application:* Standardizes client-side microphone streaming via `SpeechRecognition` (voice typing) and native system audio speech output via `SpeechSynthesis` (pronunciation guide).
    *   *URL:* [wicg.github.io/speech-api/](https://wicg.github.io/speech-api/)

*   **[R9] W3C Web Storage API (Local Persistence)**
    *   *Reference:* W3C Recommendation, "Web Storage (Second Edition)," W3C Web Applications Working Group, Apr. 2016.
    *   *Application:* Defines the `localStorage` specifications used to persist favorites lists (`dictionary_favorites`) and user queries (`dictionary_history`) directly in the browser sandbox.
    *   *URL:* [w3.org/TR/webstorage/](https://www.w3.org/TR/webstorage/)

*   **[R10] BCP-47 Locale Standardization**
    *   *Reference:* A. Phillips and M. Davis, "Tags for Identifying Languages," Best Current Practice 47, Internet Engineering Task Force (IETF), Sep. 2009.
    *   *Application:* Ensures accurate multilingual text-to-speech voicing accents by mapping input selections to standardized locale tags (e.g., `bn-BD`, `es-ES`, `fr-FR`).
    *   *URL:* [tools.ietf.org/html/bcp47](https://tools.ietf.org/html/bcp47)

## 4. External Translation & Network Protocols

This section details the third-party translation server API, HTTP client APIs, and timeout protection layers.

*   **[R11] MyMemory Translation Memory API**
    *   *Reference:* Translated SRL, "MyMemory API – Free Multilingual Translation Service," 2026.
    *   *Application:* Serves as the primary external network translation engine, providing automated contextual fallbacks for full sentences or words missing from Keya's local dictionary database.
    *   *URL:* [mymemory.translated.net/doc/spec.php](https://mymemory.translated.net/doc/spec.php)

*   **[R12] W3C Fetch API & Abort Control Standards**
    *   *Reference:* WHATWG Living Standard, "Fetch," Web Hypertext Application Technology Working Group, 2026.
    *   *Application:* Governs outbound asynchronous translation queries and standardizes network request termination via `AbortController` to enforce strict 5-second connection timeouts.
    *   *URL:* [fetch.spec.whatwg.org](https://fetch.spec.whatwg.org)
