# Use Case Descriptions

## UC-01 Search & Autocomplete
- **Primary Actor:** End User
- **Goal:** Find candidate words/phrases while typing.
- **Preconditions:** App loaded; input field available.
- **Main Flow:**
1. User types in input.
2. System runs `searchAutocomplete(query, limit)`.
3. Suggestions are shown (word/translation/phrase matches).
- **Postcondition:** Suggestion list is updated.

## UC-02 Translate Word/Phrase/Sentence
- **Primary Actor:** End User
- **Goal:** Translate text from source to target language.
- **Preconditions:** Input is not empty.
- **Main Flow:**
1. User triggers translate.
2. System resolves source language (`auto` -> detect).
3. System attempts phrase translation.
4. If phrase not found, system attempts word translation.
5. If needed, system performs word-by-word sentence translation.
6. System returns translated text and metadata.
7. System stores input in history.
- **Postcondition:** Output text and history are updated.

## UC-03 View Word Detail
- **Primary Actor:** End User
- **Goal:** View lexical data for a selected term.
- **Main Flow:**
1. User selects suggestion or translates input.
2. System runs `getWordDetail(query)`.
3. UI displays definitions, examples, synonyms, antonyms, and multi-language translations.
- **Postcondition:** Detail section is rendered.

## UC-04 Swap Languages
- **Primary Actor:** End User
- **Goal:** Reverse translation direction quickly.
- **Main Flow:**
1. User clicks swap action.
2. System determines effective source language.
3. Source/target language values are swapped.
4. Input/output text are swapped.
5. Word detail refreshes based on new input.
- **Postcondition:** Translation direction is reversed.

## UC-05 Voice Input
- **Primary Actor:** End User
- **Supporting Actor:** Speech Recognition API
- **Goal:** Fill input text using microphone.
- **Main Flow:**
1. User starts voice input.
2. System configures recognition language.
3. Browser returns transcript.
4. System sets transcript as input and translates automatically.
- **Exception:** If API unavailable, user receives an alert.

## UC-06 Speak Text
- **Primary Actor:** End User
- **Supporting Actor:** Speech Synthesis API
- **Goal:** Listen to source/translated text.
- **Main Flow:**
1. User triggers speak action.
2. System selects appropriate voice by language code.
3. Browser synthesizes speech.

## UC-07 Manage Favorites
- **Primary Actor:** End User
- **Supporting Actor:** Local Storage
- **Goal:** Save/remove words from favorites.
- **Main Flow:**
1. User toggles favorite state.
2. System updates favorites list.
3. System persists list in localStorage.

## UC-08 Manage History
- **Primary Actor:** End User
- **Supporting Actor:** Local Storage
- **Goal:** Review and clear recent lookups.
- **Main Flow:**
1. System stores each query at translation time.
2. User opens history tab.
3. User optionally clears all history.
